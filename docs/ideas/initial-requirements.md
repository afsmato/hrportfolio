# HRPortfolio - アイデアメモ

> **プロダクト名**: HRPortfolio

## このドキュメントの位置づけ

このドキュメントは壁打ち・ブレストの成果物で、正式な仕様書ではありません。

## プロダクトビジョン

HRプロフェッショナルが「自分の地図」を持ち、日々アップデートし続けるためのキャリア開発ツール。

**一言で言うと**: HRプロフェッショナルが自分のスキルを可視化・証明し、他者にアピールできる状態を作るキャリアコンパス

**北極星（このツールが目指す状態）:**
> ユーザーが「私はこれができます」と自信を持って言語化・提示できるようになること。
> 転職・社内異動・プロジェクトアサインなど、あらゆる場面で自分のスキルを根拠とともに示せる。

---

## 解決したい課題

### 今困っていること

1. **情報収集が受動的・場当たり的**
   - Googleが表示するニュースを眺める程度で、体系的に情報収集できていない
   - 「本当はトレンドや論文から学ぶべき」とわかっているが、どのソースを見ればいいかわからない
   - 学ぶべきことと日々の情報が繋がっていない

2. **スキルの地図がない**
   - HRとしてどのスキルをどの粒度で定義すべきかわからない
   - どのレベルに達していればOKという基準が自分の中にない
   - 自分の強み・ギャップを客観的に把握できていない

3. **キャリアの方向性が曖昧**
   - People Analytics・HRIS・組織開発という幅広い経験があるが、それをどう活かすか整理できていない
   - 市場でどのようなスキルが求められているか把握できていない

### 根本的な問題

```
表層の問題: 情報収集ができていない
         ↑
根本の問題: 「何を知るべきか」「何ができるべきか」の地図がない
```

## ターゲットユーザー

### ペルソナ（自分自身）

- **所属**: HR部門・組織開発担当
- **経歴**: People Analytics → HRIS → 組織開発
- **目指すキャリア**: People Analytics × 組織開発の専門家
  - データで組織を診断し、エビデンスに基づいて変革を推進できる人材
- **使い方**: 毎朝10分のチェック + 気になったときに深掘り

### 将来的なターゲット

- 同様のキャリア迷子に陥っているHR担当者全般
- HR領域でのスキルアップを目指す若手〜中堅のHRプロ

### 将来的な領域拡張

初期は People Analytics × 組織開発に特化するが、将来的にはユーザーが専門領域を選択できる設計にする。

**想定される領域:**
- People Analytics × 組織開発（初期）
- 報酬・ベネフィット（Compensation & Benefits）
- 労務・労働法（Labor Relations / Employment Law）
- タレントアクイジション（採用）
- L&D（Learning & Development）
- HRBP（HRビジネスパートナー）
- HR Tech / HRIS専門

**3つの柱:**

| 柱 | What | Why |
|---|---|---|

## スキルフレームワーク定義

AIHR・SHRMなどの外部資格フレームワークと、転職市場の求人要件を統合して定義。

### 領域1: People Analytics

| スキル | 概要 |
|---|---|
| データリテラシー | Excel・SQL・基礎統計の活用 |
| HRメトリクス設計・分析 | 離職率・エンゲージメント・パフォーマンス等の指標設計と分析 |
| データビジュアライゼーション・ストーリーテリング | データを経営・現場に伝える力 |
| サーベイ設計・分析 | エンゲージメントサーベイ・パルスサーベイの設計と分析 |
| HRIS・HRテクノロジー活用 | Workday・SAP等の活用と連携 |
| 予測分析・ワークフォースプランニング | 将来の人材需要予測・要員計画 |
| 因果推論 | 施策効果の検証・A/Bテスト設計・自然実験の活用 |
| 要因分析 | 回帰分析・多変量解析によるHR事象の要因特定 |

### 領域2: 組織開発

| スキル | 概要 |
|---|---|
| 組織診断・アセスメント | 組織の現状を多角的に診断する手法 |
| チェンジマネジメント | ADKAR・Kotter等のフレームワーク活用 |
| ファシリテーション・介入設計 | ワークショップ設計と場の促進 |
| カルチャー・エンゲージメント | 組織文化の醸成・エンゲージメント向上施策 |
| L&D設計 | 学習・能力開発プログラムの設計 |
| ステークホルダーマネジメント | 経営・現場との協働・巻き込み |

### 領域3: 戦略人事（People Analytics × OD の掛け合わせゾーン）

| スキル | 概要 |
|---|---|
| エビデンスベースドHR | データで施策を設計・検証・改善するサイクル |
| 組織有効性の測定・改善 | 組織の健全性・有効性を数値化し改善につなげる |
| エンプロイーリスニング戦略 | 多様な声の収集・分析・施策反映の仕組み設計 |
| 人材戦略・経営への提言 | データと洞察を経営判断に結びつける力 |

## 主要機能の候補

### P0（MVPで絶対必要）

#### 1. キュレーション情報フィード

People Analytics × OD 特化のソースからコンテンツを収集・表示。

**プリセットソース候補:**

| カテゴリ | ソース |
|---|---|
| People Analytics | Josh Bersin, AIHR, myHRfuture, Visier Blog |
| 組織開発 | OD Practitioner, HBR（組織・人材系）, McKinsey Org |
| HR Tech | HR Technologist, Worktech Academy |
| 国内HR | 労政時報, Works Report（リクルートワークス研究所）, HR NOTE |
| 学術・論文（HR） | SHRM研究, Academy of Management, Journal of Applied Psychology |
| 経営学 | Harvard Business Review, MIT Sloan Management Review, Strategic Management Journal |
| 労働経済学 | NBER Working Papers, IZA Discussion Papers, 日本労働研究雑誌 |

**表示イメージ:**
```
[今日のダイジェスト - 2026/03/23]

📊 People Analytics
  - "Employee Listening の最新トレンド 2026" - Josh Bersin
  - "離職予測モデルの精度向上手法" - AIHR

🏢 組織開発
  - "心理的安全性の測定アプローチ" - HBR
  - "変革期のチェンジマネジメント事例" - McKinsey

⭐ あなたへのおすすめ（スキルギャップ連動）
  - "予測分析入門" → スキル「予測分析・ワークフォースプランニング」強化に
```

#### 2. スキルマップ & 自己評価

3領域16スキルそれぞれを5段階で自己評価し、現在地を可視化。

**評価レベル定義:**
```
1: 知らない / 未経験
2: 知っている（概念理解）
3: 補助的にできる（経験あり・一人では不完全）
4: 独力でできる（実務経験あり）
5: 教えられる / 応用できる（専門性あり）
```

**表示イメージ:**
```
[スキルマップ]

People Analytics          ████░ 3.8
  データリテラシー         █████ 5
  HRメトリクス設計         ████░ 4
  データビズ・ST           ███░░ 3
  サーベイ設計             ████░ 4
  HRIS活用                █████ 5
  予測分析                 ██░░░ 2  ← ギャップ

組織開発                  ███░░ 3.2
  組織診断                 ███░░ 3
  チェンジマネジメント      ███░░ 3
  ...
```

### P1（重要だけど後回しでもいい）

#### 3. 学習キュー & 進捗管理

気になった記事・本・論文をストックし、スキルと紐付けて管理。

```
[学習キュー]
📚 積読中
  - "People Analytics入門" → 予測分析スキル
  - "大企業のOD事例" → 組織診断スキル

✅ 読了
  - "エンゲージメントサーベイ設計" → 3つのメモあり
```

#### 4. トレンドレーダー

HR領域で注目されているキーワードの変化をトラッキング。自分のスキルマップとのギャップを提示。

```
[トレンドレーダー 2026 Q1]
急上昇中のキーワード:
  🔥 AI in HR（あなたのスキルマップに未登録）
  🔥 Skills-based Organization
  📈 Employee Experience Design（一部対応済み）
```

#### 5. 読書ログ & 本・雑誌レコメンデーション

ユーザーが読んだ本を登録し、スキルと紐付けて蓄積する読書ログ機能。
加えて、書店ランキングからHR関連書籍を自動取得してレコメンドする。

**読書ログ（ユーザー起点）:**
```
[マイ読書ログ]
✅ 読了
  - "統計学が最強の学問である" → 因果推論・データリテラシー に紐付け
    メモ: 第3章の実験デザインが実務で使えそう

📚 積読
  - "ORGANIZATION DEVELOPMENT" → 組織診断・チェンジマネジメント
```

**書店ランキング連動レコメンド:**
```
[今週のHR関連ランキング - 楽天ブックス]
1位 "○○○" - ビジネス書 HR/組織カテゴリ
   → あなたの「組織診断」スキル強化に関連

2位 "△△△" - データサイエンス
   → あなたの「因果推論」スキルギャップに対応
```

**データ取得方針:**
- 楽天ブックスAPI（無料・登録制）+ Amazon PA API（アフィリエイト登録必要）の両方を利用
- ISBNで重複を排除しマージした統合書籍データベースを構築
- HR関連キーワードで自動フィルタリング

#### 6. 振り返りノート

学んだことの記録・気づきのメモ。スキルと紐付けて蓄積。

#### 7. スキルポートフォリオ（公開プロフィール）

スキルマップ・学習ログ・読書記録を「根拠付きスキル証明」として外部公開できるページ。

```
[公開プロフィール例]
──────────────────────────────
Taro Yamada｜People Analytics × 組織開発

スキルマップ
  People Analytics    ████░ 3.8
  組織開発            ███░░ 3.2
  戦略人事            ████░ 3.5

学習の根拠
  📚 読了 42冊（うちHR専門書 28冊）
  📰 記事既読 320件
  ✍️  振り返りメモ 67件

注力スキル（直近3ヶ月）
  - 因果推論：書籍3冊・記事12件
  - 組織診断：書籍2冊・記事8件
──────────────────────────────
🔗 このプロフィールを共有する
```

**活用シーン:**
- 転職活動でのポートフォリオ提示
- 社内での専門性アピール・異動申請
- HRコミュニティでの自己紹介

### P2（あったら嬉しい）

#### 8. キャリアロードマップ

現在地から目標キャリアへの道筋を可視化。次に取るべきアクションを提示。

#### 9. 資格・学習コース連携

AIHRコース・SHRM認定等と自分のスキルマップを紐付け。資格取得がスキルスコアの根拠になる。

#### 10. 他のHRプロとの比較（匿名）

同じキャリア志向を持つ人たちとスキルを比較し、相対的な立ち位置を把握。

## 技術的な検討事項（初期メモ）

### 想定する実装形態

- **フェーズ1**: Webアプリ（モバイル対応）
  - 毎朝スマホでチェックできる
  - スキル評価・学習キューの管理

- **フェーズ2**: 通知機能
  - 朝のダイジェストをプッシュ通知 or メール

### データ取得方法

- RSSフィード / WebスクレイピングでHR系ソースから記事取得
- Claude API等でタグ付け・要約・スキル連動を自動化

### スキルデータの保存

- ユーザーごとの自己評価スコア
- 学習ログ・読了記録
- 時系列でスキル成長を可視化できる形式

---

## 非機能要件（初期メモ）

- **毎日開きたくなるUI**: 情報密度が高すぎず、10分で完結できる
- **日本語優先**: 国内HRプロが主なターゲット、日本語コンテンツも充実
- **モバイルファースト**: 通勤時・隙間時間に使えること

---

## 設計上の決定事項

| テーマ | 決定内容 |
|---|---|
| スキル評価の客観性 | 現フェーズは自己評価のみ。360度評価は現プロダクトでは実装しない |
| 情報のアーカイブ | 古い記事も削除せず保管。ユーザーが増えた際にレコメンデーションエンジンの素材として活用 |
| スキルフレームワークの更新 | 3ヶ月に一度の定期棚卸しタイミングをUIで促す。フレームワーク自体もトレンドに合わせて更新 |

## 未解決の問い

1. **日本語論文・国内情報**: 海外情報と国内情報のバランスをどうとるか
2. **AIを使った学習支援**: 記事の要約・スキルマップとの紐付けをどこまで自動化するか
3. **領域拡張のタイミング**: 初期のPeople Analytics × OD以外の領域をいつ追加するか
4. **書籍検索UX**: ユーザーが本を登録するとき、タイトル入力 → ISBN自動補完の仕組みをどう作るか（楽天Books APIで検索連携が有力）
