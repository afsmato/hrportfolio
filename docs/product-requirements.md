# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**HRPortfolio** - HRプロフェッショナルが迷わず学び続けるためのキャリア学習コンパス

### プロダクトコンセプト
- **スキルの地図を持つ**: HR領域の標準的なスキルフレームワークを提供し、「何をどのレベルで知るべきか」の基準を与える
- **学習を根拠として積み上げる**: 日々の情報収集・読書ログがスキルと紐付いて蓄積され、自分の成長を実感できる
- **成長の方向性を確認する**: 蓄積された学習ログとスキルマップを振り返ることで学習の方向性を確認し、必要に応じて他者への提示にも活用できる

### プロダクトビジョン
HRプロフェッショナルが「自分は今どこにいて、次に何を学ぶべきか」を常に把握できる状態を作る。
日々の情報収集と読書が単なる習慣ではなく、スキルの成長として可視化され、学習を続けるモチベーションになる。
「何を学べばいいかわからない」「自分の成長が見えない」という迷いをなくし、HRプロとして着実に前進し続けられるツールを目指す。

### 目的
- HRとして「何を学ぶべきか」の羅針盤を提供する
- 日々の情報収集・読書を継続できる習慣形成を支援する
- 学習の積み上げを可視化し、自分の成長を実感できる環境を作る

---

## ターゲットユーザー

### プライマリーペルソナ: HR担当者（キャリアアップ志向）

**基本属性**
- 年齢: 28〜40歳
- 所属: 事業会社のHR部門
- 経験: HR歴3年以上、複数のHR職能を経験

**典型的なプロフィール**
- People Analytics・HRIS・組織開発など複数の経験を持つが、専門性の言語化ができていない
- 「HRとして次のステップに進みたい」という意欲はあるが、何をどう学べばいいか不明確
- 情報収集はGoogleニュースやSNSで受動的に行っている程度

**現在の課題**
- スキルの地図がなく、自分の強み・ギャップを把握できていない
- どのソースから情報を得ればいいか分からず、学習が体系化されていない
- 学習を続けているが、自分が成長しているかどうか実感できない

**期待する解決策**
- 自分のキャリア志向に合ったスキルフレームワークで現在地を把握したい
- 毎朝10分でHR領域のトレンドをキャッチアップできるようになりたい
- 積み上げた学習が可視化され、自分の成長を実感できるようになりたい

**1日の典型的なワークフロー**
- 朝の通勤時にスマホでダイジェストを確認（10分）
- 気になった記事・本を学習キューに追加
- 隙間時間に深掘り・振り返りメモを記録
- 3ヶ月に1度、スキルマップを棚卸し

### セカンダリーペルソナ（将来）
- 報酬・労務・採用など他のHR専門職でキャリアアップを目指す担当者
- HR領域に転職を検討しているビジネスパーソン

## 成功指標（KPI）

### 北極星指標
**学習成長実感率**: ツール利用開始から3ヶ月後に「自分の学習が積み上がっていると実感できる」と回答するユーザーが80%以上

- **測定方法**:
  - フェーズ1（MVP）: スキルマップ初回入力から90日後、最初のログイン時にサイト上でアンケートをバナー表示（ワンクリックで回答・1回のみ）
  - フェーズ2（P2: 学習サマリ実装後）: 定期サマリ生成のタイミングに合わせて毎回アンケートを実施し、成長推移を継続測定
- **対象**: アクティブユーザー（直近30日間に学習ログを1件以上記録したユーザー）
- **実装**: アンケート表示はP0機能として実装（スキルマップ & 自己評価に含める）

### プライマリーKPI

| 指標 | 目標値 | 期限 |
|---|---|---|
| 学習継続率 | アクティブユーザーの60%が週3回以上学習ログを記録 | リリース後1ヶ月 |
| スキルマップ再評価率 | アクティブユーザーの70%が3ヶ月以内に再評価を実施 | リリース後3ヶ月 |
| アクティブユーザー数 | 100人 | リリース後3ヶ月 |

※ アクティブユーザー定義は北極星指標の「対象」を参照

### セカンダリーKPI

| 指標 | 目標値 | 測定対象 | 期限 |
|---|---|---|---|
| ユーザーあたり月間学習ログ数 | 10件以上（記事・書籍合計） | アクティブユーザー | リリース後1ヶ月 |
| スキルマップ完成率 | 70%が全スキルを自己評価 | 登録ユーザー | リリース後2ヶ月 |
| スキルスコア向上率 | 60%が3領域のうちいずれか1領域の平均スコアが0.5ポイント以上向上 | 3ヶ月継続アクティブユーザー | リリース後6ヶ月 |
| アンケート回答率（参考値） | アンケート表示数の50%以上 | アンケート表示対象ユーザー | リリース後3ヶ月 |

※ スキルスコア向上率の比較基準: 初回スキルマップ完成日時点のスコアと90日後のスコアを比較する。

## 機能要件

### P0: コア機能（MVP）

#### 1. スキルマップ & 自己評価

**ユーザーストーリー**:
HRプロフェッショナルとして、自分の現在地を把握するために、標準的なスキルフレームワークに基づいて自己評価し、強みとギャップを可視化したい

**スキルフレームワーク**:
AIHR・SHRMの外部資格基準と転職市場の求人要件を統合した3領域18スキル

| 領域 | スキル |
|---|---|
| People Analytics | データリテラシー / HRメトリクス設計・分析 / データビジュアライゼーション・ストーリーテリング / サーベイ設計・分析 / HRIS・HRテクノロジー活用 / 予測分析・ワークフォースプランニング / 因果推論 / 要因分析 |
| 組織開発 | 組織診断・アセスメント / チェンジマネジメント / ファシリテーション・介入設計 / カルチャー・エンゲージメント / L&D設計 / ステークホルダーマネジメント |
| 戦略人事（People Analytics × OD） | エビデンスベースドHR / 組織有効性の測定・改善 / エンプロイーリスニング戦略 / 人材戦略・経営への提言 |

**評価レベル定義**:
- 1: 知らない / 未経験
- 2: 知っている（概念理解）
- 3: 補助的にできる（経験あり・一人では不完全）
- 4: 独力でできる（実務経験あり）
- 5: 教えられる / 応用できる（専門性あり）

**受け入れ条件**:
- [ ] 3領域18スキルを5段階で自己評価できる
- [ ] 領域ごとの平均スコアとレーダーチャートで現在地を可視化できる
- [ ] 自己評価スコアが2以下のスキルに「ギャップ」フラグが表示される（閾値は `DOMAIN_CONSTANTS.GAP_THRESHOLD` で管理）
- [ ] 評価履歴が保存され、スコアの変化を時系列で確認できる
- [ ] 最後のスキル評価から90日経過後、ログイン時にダッシュボード上にバナー通知を表示する
- [ ] 同タイミングで登録メールアドレスへ棚卸しを促すメールを送信する（Resend利用）
- [ ] 初回登録からスキルマップ全スキル入力完了までのステップ数は5段階以内とする
- [ ] 将来対応: PWA化によるプッシュ通知（現フェーズはスコープ外）

**優先度**: P0（必須）

---

#### 2. キュレーション情報フィード

**ユーザーストーリー**:
HRプロフェッショナルとして、体系的に情報収集するために、自分のキャリア志向に関連するHR領域のコンテンツが毎日キュレーションされて届くようにしたい

**プリセット情報ソース**:

| カテゴリ | ソース |
|---|---|
| People Analytics | Josh Bersin, AIHR, myHRfuture, Visier Blog |
| 組織開発 | OD Practitioner, HBR（組織・人材系）, McKinsey Org |
| HR Tech | HR Technologist, Worktech Academy |
| 国内HR | 労政時報, Works Report（リクルートワークス研究所）, HR NOTE |
| 経営学 | Harvard Business Review, MIT Sloan Management Review |
| 労働経済学 | NBER Working Papers, 日本労働研究雑誌 |
| 学術・論文（グローバル） | Journal of Applied Psychology, Academy of Management, Personnel Psychology, Human Resource Management Journal, Journal of Organizational Behavior, Journal of Applied Behavioral Science, International Journal of Human Resource Management, Journal of Labor Economics, Administrative Science Quarterly, Organization Science |
| 学術・論文（国内） | 日本労務学会誌, 産業・組織心理学研究, 経営行動科学 |
| HRコンサル（グローバル） | Deloitte Insights（Human Capital Trends）, BCG Henderson Institute, Gartner HR Research, Korn Ferry Institute |
| HRコンサル（国内） | パーソル総合研究所 |

**記事難易度タグ定義**（Claude APIによるタグ付け時に付与）:

| 難易度 | 対象スキルレベル | 記事の特徴 |
|---|---|---|
| 入門 | 1〜2 | 概念解説・用語紹介・ブログ記事 |
| 実践 | 2〜3 | 実務事例・How-to・ケーススタディ |
| 応用 | 3〜4 | 深掘り分析・研究紹介・専門メディア |
| 専門 | 4〜5 | 学術論文・最新研究・エキスパート論考 |

**受け入れ条件**:
- [ ] 毎日、各カテゴリから最新記事が自動取得・表示される
- [ ] 各記事にClaude APIで難易度タグ（入門・実践・応用・専門）が自動付与される
- [ ] スキルマップのギャップと連動した「おすすめ記事」が表示される（ギャップスキルのスコアに対応する難易度の記事を優先表示）
- [ ] 記事を「あとで読む」学習キューに追加できる
- [ ] カテゴリ・難易度ごとにフィルタリングして表示できる
- [ ] 記事の概要（100〜200字）が日本語で表示される

**優先度**: P0（必須）

### P1: 重要機能（初期リリース後3ヶ月以内）

#### 3. 学習キュー & 読書ログ

**ユーザーストーリー**:
HRプロフェッショナルとして、学習の積み上げをスキルの根拠にするために、読んだ記事・本をスキルと紐付けて記録・管理したい

**受け入れ条件**:
- [ ] 記事・書籍それぞれを「積読（未読）」「読了」で管理できる
- [ ] 各アイテムを1つ以上のスキルに紐付けできる
- [ ] 読了時に短いメモ（振り返り）を記録できる
- [ ] 読了アイテム数・メモ数がプロフィールに反映される
- [ ] 月別・スキル別に学習量を確認できる

**優先度**: P1（重要）

---

#### 4. 書店ランキング連動レコメンド

**ユーザーストーリー**:
HRプロフェッショナルとして、今読むべき本を効率的に見つけるために、書店のランキングからHR関連書籍が自動で提示されてほしい

**受け入れ条件**:
- [ ] 楽天ブックスAPIでビジネス書カテゴリのランキングを週次取得する
- [ ] Claude APIに書籍タイトル・著者・カテゴリを渡してHR関連かを自動判定し、HR関連書籍のみ表示する
- [ ] スキルマップのギャップと照合し、関連するスキルが表示される
- [ ] 書籍タイトルで検索し、自分の読書ログに追加できる（書誌情報を自動補完）

**優先度**: P1（重要）

---

#### 5. スキルポートフォリオ（個人振り返りダッシュボード）

**ユーザーストーリー**:
HRプロフェッショナルとして、自分の学習の積み上げを振り返り成長を実感するために、スキルマップ・学習ログ・読書記録を1ページで俯瞰したい。ここで言語化した自分のスキルをCVや面接でのアピールに活用する

**受け入れ条件**:
- [ ] スキルマップ・学習ログ・読書記録を1ページにまとめて確認できる
- [ ] 表示内容：スキルレーダーチャート / 読了書籍数 / 記事既読数 / 振り返りメモ数 / 注力スキル（直近3ヶ月）
- [ ] プロフィールの更新日が表示される

**優先度**: P1（重要）

---

#### 6. トレンドレーダー

**ユーザーストーリー**:
HRプロフェッショナルとして、市場でのスキルの需要変化を把握するために、HR領域のトレンドキーワードと自分のスキルマップのギャップを確認したい

**データソース**:
- フェーズ1: キュレーション情報フィード（P0）で蓄積した記事データをClaude APIで分析し、月次でトレンドキーワードを自動生成（直近2ヶ月分の記事をローリングウィンドウで分析）
- フェーズ2（将来）: 求人サイトAPIと連携し、市場で求められるスキル要件を反映

**受け入れ条件**:
- [ ] 月次でHR領域のトレンドキーワードTop10が更新・表示される（前月と同じキーワードでも更新する）
- [ ] 各キーワードに対して「自分のスキルマップとの対応関係」が表示される
- [ ] トレンドキーワードに関連する記事・書籍をワンクリックで検索できる

**優先度**: P1（重要）

---

#### 7. スキルベース名著データベース & 毎日書籍アンケート

**ユーザーストーリー**:
HRプロフェッショナルとして、記事だけでなく書籍で体系的に基礎を固めるために、「自分のスキルギャップに効く名著」を発見し、同じHRプロがどう評価しているかを参考にしながら読書計画を立てたい

**概要**:
Amazonレビューは「良い本か」を評価するが、本機能は「どのスキルが・どのレベルから伸びるか」というHRプロ視点の機能的評価を蓄積する。世界に存在しないHR特化のスキルベース書籍データベースを、ユーザーの回答から構築する。

**2段階の実装方針**:

| フェーズ | データソース | 公開条件 |
|---|---|---|
| フェーズ1（P1リリース時〜） | Claude APIでHR名著リストにスキルタグを事前付与 | 即日公開・推薦に使用 |
| フェーズ2（データ蓄積後〜） | ユーザーアンケート集計データで上書き・精度向上 | 回答数10件以上の書籍から順次切り替え |

**毎日書籍アンケートの仕様**:
- ダッシュボードに毎日1冊の名著を提示
- 回答選択肢: 「読んだ（スキル評価へ）」「積読に追加」「知らない・スキップ」
- 「読んだ」を選択したユーザーには追加質問: どのスキルが伸びたか（複数選択）・読んだ時点のスキルレベル（1〜5）
- 回答は任意・1タップで完結するUX

**受け入れ条件**:
- [ ] HR名著リスト（初期50冊以上）がClaude APIによるスキルタグ付きで登録される
- [ ] ダッシュボードに毎日1冊の名著アンケートが表示される
- [ ] 「読んだ」回答時にスキル紐付けと当時のスキルレベルを記録できる
- [ ] 「積読に追加」で学習キューに即登録される
- [ ] スキルギャップと書籍スキルタグを照合し「あなたに効く名著」が推薦される
- [ ] 各書籍の詳細ページに「このスキルが伸びると回答したHRプロ: XX人」が表示される
- [ ] 回答数が10件未満の書籍はClaude API付与タグを使用し、10件以上でアンケート集計データに切り替わる

**優先度**: P1（重要）

### P2: 将来機能

#### 8. 専門領域の選択（領域拡張）

初期はPeople Analytics × 組織開発に特化するが、将来的にはユーザーが専門領域を選択できる設計。

**想定領域**:
- 報酬・ベネフィット（Compensation & Benefits）
- 労務・労働法（Labor Relations）
- タレントアクイジション（採用）
- L&D専門
- HR Tech / HRIS専門
- HRBP

**優先度**: P2（将来対応）

---

#### 9. 学習サマリ & スキル言語化アーカイブ

**ユーザーストーリー**:
HRプロフェッショナルとして、自分の成長を実感し他者に説明できる状態を作るために、一定期間の学習内容をサマリして自分のスキルを言語化し、その履歴をアーカイブとして蓄積したい

**受け入れ条件**:
- [ ] 月次・四半期など定期的なタイミングで、その期間の学習ログ・読書記録をもとにスキル言語化サマリを自動生成する（Claude APIを利用）
- [ ] 生成されたサマリは「自分はXができる」という一人称の言語で表現される
- [ ] ユーザーはサマリを編集・保存でき、確定したものがアーカイブに追加される
- [ ] アーカイブ一覧で過去のサマリを時系列で参照でき、成長の変化を実感できる
- [ ] アーカイブのサマリは公開プロフィールに任意で掲載できる

**優先度**: P2（将来対応）

---

#### 10. 自己評価ガイド指標

**ユーザーストーリー**:
HRプロフェッショナルとして、自己評価のブレを減らしより正確に現在地を把握するために、各スキルの評価レベルに対応した具体的な行動・経験の指標を参照しながら評価したい

**受け入れ条件**:
- [ ] 各スキルの評価時に「このレベルの目安」をポップアップで参照できる
- [ ] 目安は「業務経験」「できること」「アウトプット例」の3軸で記述される（例: レベル3「サーベイ設計を上司のレビューを受けながら実施した経験がある」）
- [ ] ユーザーが過去の評価と現在の評価を見比べながら再評価できる

**優先度**: P2（将来対応）

#### 11. キャリアロードマップ

現在地から目標キャリアへの道筋を可視化し、次に取るべきアクションを提示。

**優先度**: P2（将来対応）

#### 12. 資格・学習コース連携

AIHRコース・SHRM認定等と自分のスキルマップを紐付け。資格取得がスキルスコアの根拠になる。

**優先度**: P2（将来対応）

---

#### 13. HR専門チャットボット

**ユーザーストーリー**:
HRプロフェッショナルとして、労働法・法改正やHR専門知識について即座に正確な回答を得るために、HR領域に特化したチャットボットに質問したい

**データソース（RAG）**:
- e-Gov法令API（労働基準法・育児介護休業法・労働安全衛生法等の最新法令）
- 厚生労働省通達・ガイドライン
- AIHR・SHRM等のHR専門資料（独自登録）

**料金モデル**（詳細はP2フェーズ開始時に確定）:
- トライアル: 新規登録から7日間は無制限で利用可能
- 有料プラン: トライアル終了後は月額課金制（金額はP2フェーズ開始時に設定）
- 決済インフラ: PAY.JP

**受け入れ条件**:
- [ ] HR領域の質問に対して、法令・専門資料を参照した回答を返す（Claude API + RAG）
- [ ] 回答に参照した法令・資料の出典が表示される
- [ ] 登録から7日間はトライアルとして無制限利用できる
- [ ] トライアル終了後はPAY.JPによる月額課金でのみ利用できる
- [ ] 未課金ユーザーにはアップグレードへの導線が表示される
- [ ] 法令データは定期的に最新版へ更新される（更新頻度は別途設定）

**優先度**: P2（将来対応）

#### 14. 他のHRプロとの匿名比較 & 学習行動インサイト

**ユーザーストーリー**:
HRプロフェッショナルとして、自分の学習の方向性が適切かを確認するために、同じキャリア志向・スキルレベルを持つユーザーが何を読んで成長しているかを匿名で参照し、次の学習行動の参考にしたい

**概要**:
HRPortfolioは「読んだコンテンツ × スキルレベル × キャリア属性」の三位一体データを持つ唯一のプラットフォームになる。このデータを集合知として活用することで、単なるスキル比較を超えた「成長パターンの可視化」を提供する。

**主要機能**:
- 同じキャリア志向・経験年数を持つユーザーとのスキルスコア匿名比較（相対的な立ち位置の把握）
- 特定スキルのスコアが向上したユーザーが、その直前3ヶ月に読んでいた記事・書籍の傾向表示（成長と学習の相関シグナル）
- 「自分と似た属性のユーザー」が高評価した記事・書籍のレコメンド（協調フィルタリング）

**段階的な実装方針**:
- フェーズ1（ユーザー数100人未満）: Claude APIによるコンテンツベースレコメンドをメインとして提供
- フェーズ2（ユーザー数100人以上）: 蓄積した学習行動データを加えて協調フィルタリング要素を追加
- フェーズ3（ユーザー数1,000人以上）: スキル成長パターンの分析精度が向上し、学習行動インサイトとして提供

**データ利用の前提**:
- 学習行動データの集合知利用はオプトイン制（デフォルト: 不参加）
- 個人が特定されるデータは一切公開しない（匿名集計のみ）
- オプトイン状況は設定画面からいつでも変更可能

**受け入れ条件**:
- [ ] 同じキャリア志向・経験年数帯のユーザーとのスキルスコアを匿名で比較できる
- [ ] データ利用のオプトイン・オプトアウトを設定画面から操作できる
- [ ] オプトイン時、自分と同属性のユーザーの学習傾向（人気記事・書籍Top5）が表示される
- [ ] スキルスコアが向上したユーザーの学習傾向と自分の学習ログを比較できる

**優先度**: P2（将来対応）

## 非機能要件

### パフォーマンス
- ページ表示: 2秒以内（スマートフォン・4G回線環境）
- 記事フィード取得: 1日1回バックグラウンド更新（ユーザー操作を妨げない）
- 書店ランキング取得: 週1回バックグラウンド更新

詳細なパフォーマンス目標・最適化戦略・レスポンスタイム計測条件は [architecture.md](./architecture.md) のパフォーマンス要件を参照。

### ユーザビリティ
- モバイルファースト設計（通勤・隙間時間での利用を想定）
- 新規ユーザーが初回登録からスキルマップ入力完了まで10分以内
- 毎日のダイジェスト確認が10分以内で完結するUI設計
- 日本語UI優先（コンテンツは日英混在を許容）

### 信頼性
- 学習ログ・スキルデータのRPO: 24時間（日次自動バックアップ・保持7日間）
- 将来的にはpg_dump + 外部ストレージへの定期エクスポートによる長期保管を検討

### セキュリティ
- ユーザーデータ（スキル評価・学習ログ）は非公開がデフォルト
- 公開プロフィールは明示的な操作でのみ公開される
- 外部APIキー（楽天Books API等）はサーバーサイドで管理

### スケーラビリティ
- ユーザー数が1,000人に増えてもパフォーマンス基準を維持
- 情報ソースの追加・削除が管理画面から設定可能
